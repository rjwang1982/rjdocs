## 概述

AWS 专业计划验证合作伙伴在跨行业、用例、AWS 服务和工作负载方面构建软件或提供咨询服务的专业知识。参与这些计划的合作伙伴可以提早获得新产品路线图，获得更多与 AWS 专家互动的机会，赚取经济激励，并在 AWS 上提升技能。

要获得这些权益，合作伙伴必须提供其技术专长和已证实客户成功的文档。公开案例研究是展示合作伙伴专业能力的关键组成部分。

完成的客户项目是获得专业认证的必要条件。我们鼓励合作伙伴提交公开案例研究而不是私有案例研究，因为我们使用它们来展示合作伙伴已证实的成功。例如，AWS 通过营销活动推广专业合作伙伴，将客户引导到合作伙伴解决方案查找器（PSF），在那里专业合作伙伴被优先列出，公开案例研究可以被阅读。

我们从客户和现场团队那里了解到，阅读公开案例研究让他们相信合作伙伴具有开发和交付解决方案以满足其目标所需的经验和知识。

## 案例研究要求

通过 AWS 合作伙伴中心提交案例研究时，合作伙伴必须为其发布的客户示例提供公开可用的 URL。这可以是正式的客户案例研究、白皮书、视频或博客文章。它必须包含以下信息：

### 关于客户：1 段
• 描述客户的业务和行业

### 客户挑战：1-3 段
• 描述客户的业务挑战或 IT 需求
• 解释如果不解决挑战的风险和影响

### 合作伙伴解决方案：2-6 段
• 详细说明合作伙伴如何解决客户挑战
• **重要：** 解释解决方案中使用的主要 AWS 服务以及如何交付
• **MSP 要求：** 必须包括合作伙伴在迁移前后或解决方案实施期间提供的支持服务

### 结果和效益：1-3 段
• 描述与初始客户挑战相关的解决方案结果
• **重要：** 提供解决方案效益的具体指标。例如，包括合作伙伴为客户节省的时间量、客户 IT 成本的百分比降低等

### 关于合作伙伴：1 段
• 描述合作伙伴的业务和行业
• 包括合作伙伴持有的任何现有 AWS 专业认证的提及

## 匿名公开案例研究

当合作伙伴由于参与的敏感性或安全原因无法公开命名其客户时，可以提交匿名公开案例研究。合作伙伴应提供一般客户描述（例如，客户是美国前 5 大零售商、财富 500 强金融机构等）。匿名公开案例研究可以由合作伙伴以正式客户案例研究、白皮书、视频或博客文章的形式发布。

• 请注意，合作伙伴必须在 AWS 合作伙伴中心案例研究的"公司名称"字段中提供客户名称，以便通过申请进行验证。

• **重要：** 匿名公开案例研究将由 AWS 发布到合作伙伴解决方案查找器（PSF），但客户名称将保持私密。

• AWS 合作伙伴中心将发布到合作伙伴解决方案查找器（PSF）的案例研究字段包括"案例研究标题"、"案例研究简短描述"和"案例研究 URL"。

## 专业提示

• **多重应用：** 合作伙伴可以使用同一案例研究申请多个专业认证。案例研究需要清楚地解释与所有使用它的专业认证的相关性和实施。

• **客户唯一性：** AWS 将接受每个客户一个案例研究。每个客户必须是独立的法律实体才能符合条件。如果产品对外部客户可用，合作伙伴可以使用合作伙伴内部或关联公司的示例。

• **时间要求：** **重要：** 所有案例研究必须描述在过去 18 个月内执行的部署。

• **生产环境：** 案例研究必须针对与客户处于生产阶段的项目，而不是处于"试点"或概念验证阶段。

## 公开案例研究示例

• 示例 1：人寿保险提供商利用云获得业务优势
• 示例 2：为东南亚银行服务不足人群定制的开放创新平台
• 示例 3：非洲出口银行首个基于 AWS 的核心银行基础设施

## 分步说明

按照以下步骤在 AWS 合作伙伴中心创建公开案例研究：

1. 登录 AWS 合作伙伴中心
2. 将鼠标悬停在顶部导航的"构建"上，然后点击"案例研究"
3. 点击右侧的橙色"创建"按钮
4. 提供"公司名称"（客户）、"案例研究标题"和"简短描述"
5. 在下拉菜单中，将案例研究标记为"公开"并提供"案例研究 URL"
6. 完成所有其他必填字段（标有红色星号 *）
7. 点击"提交"

**提醒：** AWS 将发布到合作伙伴解决方案查找器（PSF）的案例研究字段包括"案例研究标题"、"案例研究简短描述"和"案例研究 URL"。

有关专业申请的全面详细信息，请参阅合作伙伴中心提供的计划指南。

## 重要注意事项

### 内容要求
- 案例研究必须真实且基于实际客户项目
- 必须包含具体的量化效益指标
- 必须详细说明 AWS 服务的使用方式
- 对于 MSP：必须包括迁移前后的支持服务

### 时间限制
- 项目必须在过去 18 个月内完成
- 必须是生产环境部署，非试点项目

### 发布要求
- 必须提供公开可访问的 URL
- 可以是案例研究、白皮书、视频或博客形式
- 匿名案例研究需要在合作伙伴中心提供真实客户名称用于验证

### 质量标准
- 内容必须专业且详细
- 必须展示合作伙伴的技术能力
- 必须体现客户获得的实际价值
- 必须符合 AWS 的品牌和内容标准
